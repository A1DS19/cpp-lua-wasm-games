# The Graphics Pipeline

## The stages

| Stage                       | Runs per…  | Programmable?          | Job                                            |
|-----------------------------|------------|------------------------|------------------------------------------------|
| **Vertex shader**           | vertex     | yes (required)         | Where does this point go on screen?            |
| **Tessellation control**    | patch      | yes (optional, GL 4.0+)| Decide how finely to subdivide a patch.        |
| **Tessellation evaluation** | new vertex | yes (optional, GL 4.0+)| Produce the new subdivided vertices.           |
| **Geometry shader**         | primitive  | yes (optional)         | Emit/kill entire primitives.                   |
| **Primitive assembly**      | primitive  | no                     | Group vertices into triangles/lines.           |
| **Rasterization**           | primitive  | no                     | Which pixels does this primitive cover? Emit fragments. |
| **Fragment shader**         | fragment   | yes (required)         | What color is this fragment?                   |
| **Tests & blending**        | fragment   | no (configured)        | Does this fragment survive? Mix with existing pixel. |

Plus a separate **compute shader** (GL 4.3+) outside the render pipeline.

**In practice for jpengine-3d**, you'll write only **vertex + fragment** shaders for months. The rest are rarely needed.

## Vertex shader

A vertex shader **doesn't draw anything** — it transforms **one vertex at a time** and produces a final clip-space position.

> *"For each input vertex, output where it lands on the screen, plus any data the fragment shader will need."*

```glsl
#version 330 core
layout(location = 0) in vec3 aPos;       // one vertex in
layout(location = 1) in vec2 aUV;
out vec2 vUV;                             // passed to fragment shader

uniform mat4 uModel, uView, uProjection;

void main() {
    // model space → world → view → clip space
    gl_Position = uProjection * uView * uModel * vec4(aPos, 1.0);
    vUV = aUV;
}
```

Runs **once per vertex**. 3-vertex mesh → runs 3 times. Output: `gl_Position` plus any `out` variables.

## What happens between shaders (fixed-function)

The actual "drawing" happens **after** the vertex shader:

1. **Primitive assembly** — groups every 3 transformed vertices into a triangle (for `GL_TRIANGLES`).
2. **Rasterization** — the GPU walks the triangle and generates a **fragment** for every pixel the triangle covers.

You can't replace these with code, but you configure them via the API (winding order, cull face, polygon mode, etc.).

## Fragment shader

A **fragment** is a *candidate pixel*: a potential color for one screen location, generated by rasterization. It carries interpolated data from the vertices (UVs, normals, colors, etc.).

> *"Given this fragment's interpolated data, compute its color."*

```glsl
#version 330 core
in vec2 vUV;
out vec4 FragColor;

uniform sampler2D uTexture;

void main() {
    FragColor = texture(uTexture, vUV);
}
```

Runs **once per fragment** — potentially millions of times per frame. A triangle covering 100,000 pixels → the fragment shader runs 100,000 times for it. That's why fragment shaders are the usual performance bottleneck.

## Fragment vs pixel — the distinction

A fragment is not a pixel **yet**. Whether it becomes an actual pixel depends on what happens after the fragment shader:

| Test / stage      | Purpose                                                             |
|-------------------|---------------------------------------------------------------------|
| **Depth test**    | Is something closer already drawn here? If yes, discard.            |
| **Stencil test**  | Masked out by the stencil buffer?                                   |
| **Blending**      | Merge with existing framebuffer pixel (transparency).               |
| **Scissor / write masks** | Culled / channel-masked.                                    |

If the fragment survives all of these, its color is written to the framebuffer — **then** it becomes a pixel.

## Frame mental model

Every frame, for every triangle:

```
vertex shader  →  primitive assembly  →  rasterization  →
fragment shader  →  depth/stencil  →  blending  →  framebuffer
```

Per-vertex work happens once per vertex. Per-fragment work happens once per covered pixel. Per-primitive work (assembly, clipping, face culling) happens once per triangle.
